About Shipment Tracker

Shipment Tracker is a modular Angular application designed to demonstrate best practices in architecture and code quality for modern web development.

Project Structure & Architecture

  • Modular Architecture: Features are organized into dedicated folders such as core, shared, features, layouts, models, utils, and assets for scalability and maintainability.
  • Feature Modules: Each major feature (e.g., Shipments) is encapsulated in its own folder under features/ with components, services, data, and models grouped together.
  • Shared Components: Common UI and utility components are placed in shared/components for reuse across the app.
  • Absolute Imports: TypeScript path mappings enable clean and maintainable imports.
  • Standalone Components: Angular standalone components are used for improved modularity.

Code Quality

  • Strict Linting: ESLint (flat config) is used with Angular and TypeScript plugins to enforce consistent code style and catch errors early.
    To run ESLint:
    npx eslint .
  • Formatting: Prettier ensures consistent formatting across the codebase.
    To format code:
    npx prettier --write .
  • Pre-commit Hooks: Husky and lint-staged run linting and formatting checks before every commit.
    Pre-commit checks run automatically on git commit:
    git commit -m "your message"
  • Unit Testing: Jasmine and Karma are set up for comprehensive unit testing of components and services.
    To run unit tests:
    npm test
  • Mock Data: The app uses mock shipment data for demonstration and testing purposes.
